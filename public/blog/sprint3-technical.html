An analogy to describe the differences between HTML and CSS

HTML is responsible for the content of a website, including the text, images, and links. It describes the structure of the website. CSS describes the presentation of the content of a website, how the HTML document should appear when rendered by a browser.

An analogy to show the differences between HTML and CSS is a person's body. HTML describes the structure and content of their body, that it consists of a skeleton, musculature, and organs. CSS describes how they actually look, from details as visible as height, eye, and hair colour down to fine details like facial features.

Explain control flow and loops using an example process from everyday life

We can understand control flow as being the order in which a series of instructions is carried out. In a computer program these are code statements being executed, but we can see control flow in more mundane processes, like making tea (which I'm probably doing as you're reading this).

  1. Get mug and teapot, put teabags into teapot.
  2. Fill kettle with water, and set to boil.
  3. Wait until water boils.
  4. Pour boiled water to fill teapot
  5. Wait for tea to steep.
  6. Fish out teabags, trying not to scald fingers.
  7. Pour tea into awaiting mug.
  8. Add sugar to taste, and enjoy tea.

In the process above, at multiple points an action is being repeated until a condition is met: waiting until the water boils, removing teabags while teabags remain in the teapot, etc. This is referred to as loop in programming.

Describe what the DOM is and an example of how you might interact with it.

The Document Object Model, or DOM, represents an HTML document as a tree structure where each node is an object that represents a part of the document. It allows programs to interact with the structure, style, and content of the document through the nodes of the tree. One way one might interact with the DOM, is to use JavaScript to change the properties of an element.

                        
                            .blue {
                                color: blue;
                            }

Assuming the above CSS class this following JavaScript example adds the class .blue to the first paragraph element of the document.

                        
                            document.getElementsByTagName("p")[0].classList.add("blue");

Explain the difference between accessing data from arrays and objects.

An object is one of the base datatypes in JavaScript. It is a list of properties, which each consist of a colon-separated key-value pair where the key is a string name, and the value can be any piece of data.

                        
                            let obj={
                                "name": "Chris",
                                "age": 26,
                                "likes": ["cats", "coding", "pizza"],
                            };

Data can be accessed from an object using bracket or dot notation by specifying the key, or property name. When using the dot notation the quotes around the property name are omitted.

                        
                            console.log(obj.name); // Chris
                            console.log(obj["age"]); // 26

An array is a specialised object in JavaScript that is an ordered collection of data. Arrays differ from base objects in that they have an order, and are not accessed with string keys. Instead arrays use numbers as the keys, starting from 0.

                        
                            let arr=["cats", "coding", "pizza"];

Data in an array can only be accessed via bracket notation by specifying the key, more commonly known as the index.

                        
                            console.log(arr[0]); // cats

Explain what functions are and why they are useful.

A function is one of the core components of a program. It is basically, a named collection of code. When a programmer defines a function they specify a name, an optional list of comma-separated parameters, and then the code of the function enclosed between curly braces.

                        
                            function sayHello() {
                                console.log("Hello world!"); // Hello world!
                            }
                            
                            function sum(x, y) {
                                return x + y;
                            }
                            console.log(sum(2, 7)); // 9

These can then be "called" from elsewhere in the program, and reused as necessary. When called the flow of the program will pause for the function's code to run, and then continue from where it left off. Grouping code into functions is incredibly useful. In addition to reusability, they allow the programmer to repeat themself less, give them a way to organise a program, and make editing a program much easier.
